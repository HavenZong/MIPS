# 三级评测测试说明

本测试用于验证 32 位 MIPS CPU 对 MIPS-C3 指令集、基础版监控程序和串口交互流程的支持情况。

## 指令要求

三级评测要求支持以下 21 条指令：

- `and`
- `andi`
- `lui`
- `or`
- `ori`
- `xor`
- `xori`
- `sll`
- `srl`
- `addu`
- `addiu`
- `beq`
- `bgtz`
- `bne`
- `j`
- `jal`
- `jr`
- `lb`
- `sb`
- `lw`
- `sw`

基础版监控程序源码说明中还列出过 `mul`，当前 CPU 已实现该指令，因此即使官方 kernel 使用它也能执行。

## CPU 与外设要求

- 虚拟内存空间为 `0x80000000` 到 `0x807FFFFF`，共 8MB。
- `0x80000000` 到 `0x803FFFFF` 映射到 BaseRAM。
- `0x80400000` 到 `0x807FFFFF` 映射到 ExtRAM。
- CPU 使用小端序。
- CPU 复位后从 `0x80000000` 开始取指执行。
- `reset_btn` 高电平时 CPU 复位，松开后解除复位。
- 当前设计使用 `clk_50M`。
- 串口为 9600 波特率，8 数据位，1 停止位，无校验。

## 串口 MMIO 映射

基础版监控程序访问以下地址：

| 地址 | 位 | 说明 |
| --- | --- | --- |
| `0xBFD003F8` | `[7:0]` | 串口数据口，读表示接收一个字节，写表示发送一个字节 |
| `0xBFD003FC` | `[0]` | 只读，为 1 表示串口空闲，可发送 |
| `0xBFD003FC` | `[1]` | 只读，为 1 表示串口收到数据 |

本项目的 `src/soc/soc_bus.v` 使用同一组地址和状态位。

## 自动化测试流程

1. 将拨码开关设置为 0。
2. 将 `kernel.bin` 下载到 BaseRAM 起始位置。
3. 单击复位按钮。
4. CPU 从 `0x80000000` 启动基础版 Kernel。
5. Term 通过串口等待 Kernel 输出欢迎信息。
6. Term 使用 `A` 命令将用户程序加载到 `0x80100000`。
7. Term 使用 `D` 命令读出用户程序，检查加载结果。
8. Term 使用 `G` 命令执行用户程序。
9. Term 使用 `R` 和 `D` 命令读取寄存器和内存，检查执行结果。

## 本地测试

官方下载文件：

```text
/home/luoshuang/下载/kernel.bin
/home/luoshuang/下载/supervisor_mips.zip
```

本仓库新增的三级用户程序：

```text
asm/c3-user.s
```

该用户程序链接到 `0x80100000`，通过监控程序 `A` 命令加载，执行后将失败计数写到 `0x80400000`。值为 0 表示通过。

构建命令：

```sh
make -B -C asm c3-user.bin
```

本地 Kernel/串口交互仿真命令：

```sh
make -C sim kernel-c3
```

完整基础回归仍可使用：

```sh
make -C sim all
```
